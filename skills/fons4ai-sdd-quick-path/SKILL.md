# Fons4AI SDD Quick Path (S0)

## Contract

### Inputs

- Required: 用户的变更请求，且变更范围满足 S0 准入条件（见下方"S0 准入条件"）。
- Optional: `AGENTS.md`、`.specify/memory/index.md`、项目规则、受影响文件的当前内容。
- Forbidden assumptions: 不得把涉及业务逻辑、数据模型、公共契约、安全权限或跨模块影响的变更降级为 S0；不得从用户泛化表达推断 S0 适用。

### Preconditions

- Entry gates: 满足触发条件之一，且变更满足 S0 准入条件的全部条目。
- Required source artifacts: 必须先读取受影响文件的当前内容，确认变更范围不触碰 S1/S2 触发条件。
- Escalation gate: 如果读取后发现变更不满足 S0 准入条件，必须停止并建议转入 `fons4ai-sdd-feature-workflow`（S1/S2）或 `fons4ai-sdd-change`。

### Outputs

- May create or update: `spec/quick/<yyyymmdd>/<变更中文名>-快速变更记录.md`；用户确认实现后可修改 `Files:` 指定的业务代码或配置文件。
- Must not create or update: SDD 需求说明书、技术设计说明书、任务规划、CR、实施报告、BUG 修复报告、知识库正文、`.specify/sql/`、执行型 DDL。
- Output location: `spec/quick/<yyyymmdd>/`。

### Exit Criteria

- Success: 快速变更记录已生成并通过 S0 校验；用户确认实现后变更已完成、验证通过、结果已回写快速变更记录。
- Blocked: 变更不满足 S0 准入条件、用户未确认实现、验证失败、或实现过程中发现需要升级到 S1/S2。
- Failure report: 汇报阻塞原因、已生成产物、建议转入的技能和用户需要回答的问题。

### Handoff

- Next skill: 默认无自动下游；如变更范围扩大，建议转入 `fons4ai-sdd-feature-workflow`（新功能）或 `fons4ai-sdd-change`（已有功能迭代）；如发现是 BUG，转入 `fons4ai-bugfix-workflow`。
- Required handoff fields: 快速变更记录路径、S0 等级、变更范围、受影响文件、验证方式、验证结果、实现确认状态。
- Stop condition: 快速变更记录生成后必须停止，等待用户确认实现；不得自动进入实现。

## 触发门禁

使用本技能前，必须确认至少满足以下任一条件：

1. 用户明确指定该技能，例如 `$fons4ai-sdd-quick-path`。
2. 用户明确要求轻量处理、快速变更或指定 S0 等级。
3. 当前仓库作用域内存在启用路由的 `AGENTS.md`，且用户请求明显属于 S0 准入范围内的低风险小变更。

如果以上条件都不满足，不得自动应用本技能。应继续使用普通 AI agent 行为，或建议用户选择 S1/S2 工作流。

## S0 准入条件

变更必须**同时满足全部**以下条件，才可使用 S0：

1. 影响文件数不超过 5 个，且属于同一模块或紧密相关的配置/资源文件。
2. 不改变业务逻辑、业务规则或业务状态流转。
3. 不改变数据模型、表结构、索引、约束或数据服务结构（Redis/MQ/ES 等）。
4. 不改变公共 API、RPC 契口、消息契约或对外接口签名。
5. 不改变权限、安全、认证、授权或风控逻辑。
6. 不涉及跨核心模块影响。
7. 不涉及资金、支付、库存、对账或监管数据。
8. 变更可独立验证，验证方式明确且可快速执行。

典型 S0 场景：

| 场景 | 说明 |
| --- | --- |
| 文案修改 | UI 标签、提示语、错误消息、注释 |
| 样式调整 | CSS、布局、间距、颜色 |
| 配置值变更 | 阈值、超时时间、开关、特性标志 |
| 依赖版本升级 | 不引入破坏性变更的版本 bump |
| 纯重命名/格式化 | 不改变行为的重构 |
| 文档更新 | README、API 文档、注释文档 |
| 日志级别调整 | 日志级别、日志格式 |

如果变更不满足全部准入条件，必须升级到 S1 或 S2。

## 角色说明

你是轻量变更的执行者，负责在低风险小范围变更中提供足够的过程纪律，但不引入完整 SDD 的开销。

你的核心判断是：

- 这个变更是否真的满足 S0 准入条件？
- 变更范围、受影响文件和验证方式是否清晰到可以安全执行？
- 实现过程中是否发现需要升级？

## 职责边界

本技能负责：

- 判断变更是否满足 S0 准入条件。
- 生成快速变更记录（包含范围、受影响文件、风险判断和验证方式）。
- 在用户确认后执行变更。
- 运行验证并回写结果。
- 发现范围扩大时停止并建议升级。

本技能不得：

- 跳过确认门禁直接实现。
- 跳过验证直接声明完成。
- 处理 S1/S2 范围的变更。
- 在实现过程中扩大范围而不升级。
- 修改 SDD 产物、知识库或 SQL 文件。
- 生成需求说明书、技术设计说明书或任务规划。

## 必要上下文

1. `AGENTS.md`（如存在）。
2. 受影响文件的当前内容。
3. 相关项目规则（如存在）。
4. 不全量读取知识库、规则或历史 spec。

## 执行流程

### 1. 判断 S0 准入

读取用户请求和受影响文件，逐条检查 S0 准入条件。

- 全部满足：继续。
- 任一不满足：停止，建议转入 S1/S2 或 BUG 工作流，说明不满足的条目。

### 2. 生成快速变更记录

使用 `assets/templates/quick-change-record-template.md` 生成：

```text
spec/quick/<yyyymmdd>/<变更中文名>-快速变更记录.md
```

记录内容至少包含：

- 变更说明（一句话）。
- 变更原因。
- 受影响文件清单。
- S0 准入自查表（逐条确认）。
- 验证方式。
- 实现确认门禁（`pending`）。

### 3. 运行 S0 校验

Python 可用时运行：

```text
python <fons4ai-sdd-quick-path>/scripts/validate_quick_change.py --report <快速变更记录路径>
```

校验失败时补齐缺失字段，再继续。

### 4. 停止等待确认

明确说明：下一步只有在用户回复 `执行`、`确认实现` 或 `继续` 后，才能进入实现。

不得自动进入实现。

### 5. 执行变更

用户确认后：

- 只修改 `Files:` 指定的文件。
- 实现过程中如发现需要修改超出 `Files:` 范围的文件，或发现变更触及 S1/S2 触发条件，必须停止并建议升级。
- 实现完成后运行验证。

### 6. 验证并回写结果

- 运行快速变更记录中声明的验证方式。
- 验证通过：回写验证结果、变更文件清单和完成状态到快速变更记录。
- 验证失败：保留未完成状态，报告失败原因和修复建议。

### 7. 升级处理

如果任一阶段发现变更不满足 S0 准入条件：

- 停止当前流程。
- 在快速变更记录中标记 `状态：需升级`。
- 说明不满足的准入条目。
- 建议转入 `fons4ai-sdd-feature-workflow`（S1/S2）或 `fons4ai-sdd-change`。

## 质量要求

- 快速变更记录必须通过 S0 校验脚本。
- 变更完成后必须有验证结果（测试命令输出、构建结果或手工验证记录）。
- 验证失败时不得标记完成。
- 变更文件清单必须与实际修改的文件一致。

## 禁止事项

- 不得把 S1/S2 范围的变更降级为 S0。
- 不得跳过确认门禁。
- 不得跳过验证。
- 不得在实现过程中扩大范围而不升级。
- 不得修改 SDD 产物、知识库或 SQL 文件。
- 不得把"看一下""继续看看"当作实现授权。

## 结束回复要求

完成后必须说明：

- 做了什么变更。
- 快速变更记录路径。
- 变更文件清单。
- 验证方式和验证结果。
- 是否有未验证项或升级风险。
- 下一步建议（如需要知识汇总或后续工作）。
