# Agent 运行规则参考

使用本参考创建或更新 `.specify/rules/agent运行规则.md`。

该规则文件是项目级 AI agent 执行约束，不是详细架构文档，也不是完整开发手册。它应在 100 行左右保持可读，优先写必须遵守的行为边界。

## 默认文件

默认只生成：

- `.specify/rules/agent运行规则.md`

当用户要求生成代码编写规范、代码风格约束、API 设计约束或编码质量规则时，额外生成：

- `.specify/rules/代码编写规范.md`

当用户要求生成 SDD 团队协作规范、企业级 SDD 准入规则、评审门禁或完成定义时，额外生成：

- `.specify/rules/sdd团队协作规范.md`

不要再生成旧的多文件规则集。

如果旧规则文件已存在，可以把它们作为历史输入读取，但不得自动更新、删除或重建。

## 信息来源

按需读取：

- `AGENTS.md`
- `.specify/memory/index.md`
- 命中的领域文档和知识卡片
- 已有 `.specify/rules/agent运行规则.md`
- 已有 `.specify/rules/代码编写规范.md`
- 旧规则文件或团队自定义规则
- 代表性的源码、测试、配置、构建文件
- 用户明确提供的约束

不要为了生成规则而全量读取所有知识库、源码或文档。

## 必须表达的规则主题

`agent运行规则.md` 必须覆盖：

- 修改前理解需求；
- 优先复用已有代码；
- 优先遵循项目规范；
- 不修改无关代码；
- 不擅自引入新框架；
- 不删除核心业务逻辑；
- 不擅自修改数据库结构；
- MCP 与外部工具使用规则：默认只写通用安全边界；只有存在项目级 MCP 配置或用户明确要求时，才写 MCP 工具清单；
- 修改后的输出内容；
- 禁止猜测业务逻辑、编造接口、编造数据库字段、编造第三方 API；
- 信息不足时主动说明缺失信息。

## 代码编写规范主题

`代码编写规范.md` 只约束 AI agent 编写或修改代码时的注意事项，不记录项目技术栈、模块结构、数据库事实或业务事实。

必须覆盖：

- 基本编码原则；
- 工具类与已有代码复用；
- 代码风格和可读性；
- DDD-lite 编码约束；
- API 接口设计；
- 异常与日志；
- 数据访问与事务；
- 数据映射与安全：字段映射、金额单位、日期格式、状态口径、外部流水、客户标识、敏感数据、日志脱敏、加密、权限、审计；
- 测试与验证；
- 禁止事项。

不得把知识库中已经存在的项目技术栈、架构说明、领域说明、数据库结构搬进该文件。

## SDD 团队协作规范主题

`sdd团队协作规范.md` 只约束团队如何使用 SDD，不记录具体项目需求、技术方案或业务事实。

必须覆盖：

- SDD 使用准入规则：哪些场景必须走 SDD、哪些可以轻量处理、哪些必须 S2；
- SDD 产物质量标准：需求说明书、技术设计说明书、任务规划分别什么算合格；
- 数据设计与治理门禁：数据影响判断、字段映射契约、数据流设计、数据安全与合规、样例验证；
- 角色与评审门禁：需求、设计、任务、S2、DDL、安全等场景由谁评审；
- 评审结论：通过、有条件通过、退回修改、暂缓；
- 标准扩展场景与专业工作流复用：已有标准能力扩展时如何复用专业工作流，并限制实现阶段自由发挥；
- 标准样例库引用：只引用 `.specify/examples/sdd/`，不复制样例正文；
- SDD 完成定义：实现、验证、交付说明、知识汇总判断和风险记录的闭环。

不得把该文件写成 SDD 教程，不得复制具体功能的三件套内容，不得把样例当成项目事实。

## MCP 与外部工具规则边界

可以在 `agent运行规则.md` 中增加 MCP 相关规则，但默认只写通用安全边界，不生成 MCP 表格。

- 只有仓库内存在项目级 MCP 配置文件、团队规范文件、AGENTS.md/知识库明确声明统一 MCP 能力，或用户明确要求把指定 MCP 写入项目规则时，才可以生成 MCP 工具清单或表格。
- 个人本机 MCP 能力不是项目规则来源，不得写成团队规范。
- 没有项目级 MCP 配置时，只保留 MCP 使用原则，不生成 MCP 表格，不得用 `待确认` 表格占位。
- 没有事实时，不得编造 MCP 名称、数据库、账号、环境或连接方式。
- 默认优先只读。任何写入、删除、更新、发送请求或影响外部系统的 MCP 操作都必须先获得用户明确确认。
- 多个 MCP、数据库、环境或账号都可能匹配时，必须先询问用户选择。
- 不得把查询语句、连接信息、令牌、账号、内部地址、个人路径或敏感返回内容写入长期文档，除非项目明确要求且已脱敏。
- 若生成 MCP 表格，只能包含服务名、项目用途、配置来源、默认权限、使用边界和状态等项目级事实。

## 知识库未初始化的项目处理

部分项目可能还没有 `.specify/memory/` 或 `.specify/rules/`。这种情况下可以先创建 `agent运行规则.md` 作为最小执行约束。

规则必须避免伪造项目事实。没有证据的技术栈、模块、数据库、接口和 CI 流程都应写为 `待确认`，或不写入规则。

## 旧规则迁移

如果旧规则文件已经存在，生成器可以建议迁移：

- 将通用执行约束合并到 `agent运行规则.md`；
- 将详细代码风格、结构、测试、数据规则保留为历史参考或后续重构对象；
- 删除旧规则前必须获得用户确认。
