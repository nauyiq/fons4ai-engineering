# 领域知识建模逐问确认模板

> 用途：正式生成领域文档前，在对话中逐个确认关键问题。
> 约束：不得只把问题写入文档后继续生成正式领域文档。

## 1. 输出规则

- 默认逐问逐答，每次只输出一个当前最高优先级问题。
- 用户回答当前问题后，先记录确认结论，再提出下一个问题。
- 不得一次性输出多个确认问题，除非用户明确要求“批量列出问题”“一次性给出所有问题”或“全部按推荐”。
- 没有剩余阻塞问题后，输出确认摘要，再生成正式领域文档。
- 普通待补充信息写入领域文档“待确认事项”，不要阻塞领域建模。

## 2. 问题格式

```text
Q1/N：<需要确认的问题>
推荐：<推荐答案>
理由：<证据和架构判断>
可选：A. <方案一>；B. <方案二>
影响：<影响的领域文档、知识卡片、标准流程或后续建模>
请回复：确认 / 按推荐 / 改为 <你的结论> / 跳过，写入待确认事项
```

## 3. 优先确认的问题

| 优先级 | 问题类型 | 判断标准 |
| --- | --- | --- |
| P0 | 领域名称与 slug | 影响目录、索引、知识卡片归属 |
| P0 | 领域边界 | 影响能力、对象、数据和流程是否纳入本领域 |
| P0 | 标准流程判定 | 影响是否把公共抽象写成标准，或把单实现降级为代表性实现 |
| P1 | 核心业务能力 | 影响业务文档和技术落地范围 |
| P1 | 适配方案 | 影响是否需要适配矩阵、适配说明或证据账本 |
| P2 | 待补充细节 | 不影响三文档，可写入待确认事项 |
