# Fons4AI 技能验证矩阵

> 用途：记录每个技能在不同项目类型中的验证状态，驱动技能成熟度判定。
> 更新规则：每完成一个试点项目，更新对应技能的项目验证状态。

## 验证状态说明

| 状态 | 含义 |
|------|------|
| 未验证 | 尚未在任何项目中实际使用 |
| 试点中 | 至少一个项目正在使用，反馈未闭环 |
| 已验证（单项目） | 一个项目完成完整链路，反馈已闭环 |
| 已验证（多项目） | 至少两个不同类型项目完成完整链路，无阻塞问题 |
| 稳定 | 三个以上项目验证，可作为推荐入口 |

## 待验证项目类型

| 项目类型 | 优先级 | 原因 |
|---------|--------|------|
| 业务系统（Java/Spring） | 高 | 最常见的项目类型，验证 SDD 主链路在业务场景中的适用性 |
| 业务系统（Python/Go） | 中 | 验证技术栈适配需求 |
| 前端项目 | 中 | 验证技能对非后端项目的适用性 |
| 混合项目 | 低 | 边界场景，优先级低于纯业务和纯技术项目 |
